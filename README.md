# Pizza-Sales-Analysis-SQL-Project-
**PROJECT OVERVIEW**

This project focuses on analyzing pizza sales data using SQL to uncover meaningful business insights. By working with structured datasets, the project demonstrates how SQL queries can be used to explore customer behavior, sales trends, and revenue generation.

The goal is to understand how a pizza business operates by analyzing orders, pizza types, sizes, quantities, and pricing data.

**OBJECTIVES**

1. Analyze overall sales performance
2. Identify best-selling pizza and categories
3. Understand customer ordering behavior
4. Evaluate revenue generation
5. Optimize business decision-making using data insights

**DATABASE SCHEMA**

The dataset consists of the following tables:

1. **orders :** Stores order IDs, dates, and times
2. **orders_details :** Includes order IDs, pizza IDs, and quantity
3. **pizzas :** Stores pizza IDs, type, size, and price
4. **pizza_types :** Contains pizza name, category, and ingredients

**KEY BUSINESS QUESTIONS**

1. Total Number of Orders
2. Total Revenue Generated
3. Top 5 Most Ordered Pizza Types
4. Quantity by Pizza Category
5. Order Distribution by Hour
6. Category-wise Pizza Distribution
7. Highest Priced Pizza
8. Most Common Pizza Size

**KEY INSIGHTS**

1. Large-sized pizzas are the most commonly ordered
2. Certain pizza types dominate sales volume
3. Peak order hours occur during midday and evening
4. Classic and Supreme categories generate high demand
5. Revenue insights help identify high-value menu items

**TOOLS & TECHNOLOGIES USED**

1. SQL (MySQL)
2. Relational Database Management Systems
3. Data Analysis Techniques

**CONCLUSION**

This project highlights how SQL can be effectively used to analyze sales data and extract actionable insights. It provides a strong foundation for data analytics, business intelligence, and decision-making in real-world scenarios.
